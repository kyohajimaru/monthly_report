# Googleスプレッドシート月次レポート自動化ツール

GA4、Search Console、問い合わせ管理シートを月次集計し、Googleスプレッドシート上のメニューからレポートを作成するApps Scriptです。

## ファイル

- `monthly_report.gs`: Apps Scriptに貼り付けるコード本体

## 初回セットアップ

1. Googleスプレッドシートを新規作成、または既存の管理用スプレッドシートを開きます。
2. `拡張機能 > Apps Script` を開きます。
3. `monthly_report.gs` の内容をApps Scriptエディタに貼り付けます。
4. Apps Scriptエディタ左側の `サービス` から以下を追加します。
   - `Google Analytics Data API`、識別子: `AnalyticsData`
   - `Google Search Console API`、識別子: `SearchConsole`
5. 初回実行時にGoogle Cloud側のAPI有効化を求められた場合は、案内に従って有効化します。
6. Apps Scriptで `setupMonthlyReportSheets()` を1回実行します。
7. スプレッドシートに戻り、`設定` シートを入力します。

## 設定シート

| 項目 | 入力例 | 補足 |
| --- | --- | --- |
| サイト名 | 京都はじまるサポート | レポート見出しに使います |
| GA4プロパティID | 123456789 | GA4管理画面のプロパティID。数値のみ |
| Search ConsoleのサイトURL | https://example.com/ | Search Console登録プロパティと完全一致させます |
| 問い合わせ管理シート名 | 問い合わせ | 問い合わせ一覧が入っているシート名 |
| レポート対象月 | 2026-06 | `yyyy-MM` 形式 |

Search Consoleがドメインプロパティの場合は、`sc-domain:example.com` の形式で入力します。

## 問い合わせ管理シートの前提

問い合わせ数は、指定した問い合わせ管理シートの1行目から日付列を探して集計します。

日付列の見出しは、以下のいずれかを含めてください。

- `日付`
- `日時`
- `受付日`
- `送信日時`
- `created_at`
- `created`
- `date`

対象月内の日付が入っている行数を問い合わせ数としてカウントします。

## 使い方

スプレッドシートを再読み込みすると、上部メニューに `月次レポート` が追加されます。

- `今月分を作成`: 今日が属する月のレポートを作成
- `前月分を作成`: 前月のレポートを作成
- `初期シートを作成`: 必要なシートを再作成

作成されるシートは以下です。

- `設定`
- `月次サマリー`
- `検索キーワード`
- `人気ページ`
- `レポート`

`レポート` シートはPDF出力用です。Googleスプレッドシートの `ファイル > ダウンロード > PDF` からPDF化できます。

## 注意点

- 初回実行時はGoogleアカウントの承認が必要です。
- GA4、Search Consoleともに、実行ユーザーが対象プロパティを閲覧できる必要があります。
- Search Consoleのデータは反映まで数日遅れることがあります。
- Clarity連携は含めていません。
